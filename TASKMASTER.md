## Architecture

```
index.html              — Single-page portfolio (hero, projects, dataviz, hobbies, about)
style.css               — All styles (glass-morphism theme, dark background)
projects_config.json    — Curated metadata for GitHub repos (category, display name, description, tech pills)
update_projects.py      — Python script (stdlib only) that fetches live repo data and rebuilds project cards
.github/workflows/update-projects.yml — GitHub Action: runs on config push + daily + manual trigger
resources/              — Images, thumbnails, PDF CV
README.md               — User-facing guide for adding projects
.gitignore              — Hides litellm_config.yaml, TASKMASTER.md, .claude/
```

## Key mechanics

- **Projects section** is auto-generated by `update_projects.py`. It reads `projects_config.json`, fetches live repo data from the GitHub API, and replaces everything between `<!-- PROJECTS_START -->` and `<!-- PROJECTS_END -->` in `index.html`.
- **Adding a project:** Edit `projects_config.json` on GitHub's web UI (pencil icon → add entry → commit). The Action triggers on any push to `projects_config.json` and rebuilds within ~30 seconds. No cloning needed.
- **Scrollable projects grid** uses `max-height: 640px` + `overflow-y: auto` with a CSS `mask-image` fade at the bottom edge (shows 6 cards / 2 rows on desktop before scrolling).
- **Photo/hobbies grid** uses a 12-column CSS grid with `content-visibility: auto` + `contain: layout style paint` for performance. GPU-heavy properties (`will-change`, `backface-visibility`, `translateZ`) were deliberately removed from the default image state to prevent scroll lag.
